Getting Started

Getting Started with NPM

Getting up and running with TickTock via NPM and Node.js is quick. First, install the package into your project.

yarn add ticktock-js --dev

Include the package in your project. 

const Watch = require('ticktock-js');

Lastly, create the settings object and instantiate the new Watch class. 

const settings = {
  dials: [{
    name: 'primary',
    hands: {
      hour: 'primary-hour-hand',
      minute: 'primary-minute-hand',
      second: 'primary-second-hand'
    }
  },
  {
    name: 'secondary',
    hands: {
      hour: 'secondary-hour-hand',
      minute: 'secondary-minute-hand',
      second: 'secondary-second-hand'
    },
    offset: '+4',
    sweep: true
  }],
  reserve: {
    id: 'power-reserve-hand',
    range: [-90, 90]
  }
};
let demo = new Watch(settings);

Calendars

It is very common to find watches and clocks that display calendar information. Whether this is a simple date display or a full perpetual calendar which shows the month, date, day, and a year indicator to show the current year's relation to a leap year. TickTock supports all of these individual complications so any combination of such can be created.

Date - Overview
Property: date
Type: Object
Required: False
Default: null

Date indicators are used to show the current date of the month. They're commonly paired with a Day Indicator and part of several date-related complications comprising annual and perpetual calendars.

TickTock expects the initial position of the indicator to be on the 1st. It will then rotate for each day from there. 

let settings = {
  ...
  date: {
    id: 'date-disc'
  }
};
Date - Define the Indicator
Property: id
Type: String
Required: True

The date object accepts an id property. This property expects a string value of the indicator element's ID. The indicator's initial position is expected to be on the 1st and will rotate clockwise for each day from there.

let settings = {
  ...
  date: {
    id: 'date-disc'
  }
};
Date - Defining a Split Display
Property: split
Type: Object
Required: False
Required: null

Some date displays are broken into two split discs. One disc shows the ones of the date ranging from 0 - 9 while the other shows the tenths, 0 - 3. Combined, these two split discs display the full date.

TickTock supports this functionality with the split object. By defining the IDs of the ones and tenths discs, TickTock will handle both discs simultaneously. Both discs are expected to be initially placed in their 0 position.

Note that the Date property accepts either an id or split property. Including both will result in an error. 

let settings = {
  ...
  date: {
    split: {
      ones: 'ones-disc',
      tenths: 'tenths-disc'
    }
  }
};
Day - Overview
Property: day
Type: Object
Required: False
Default: null

The Day Indicator will display the name of the current day of the week. Commonly, this component is built as a rotating disc, as in the demo, but can also be represented with a rotating hand.

The default behavior of TickTock is to rotate the day disc once every day to show the correct label beginning with Sunday. However, an additional setting, offsetHours can be used to rotate the element an additional amount for the current hour to visually represent the day's progress. 

let settings = {
  ...
  day: {
    id: 'day-indicator-disc',
    offsetHours: true
  }
};
Day - Define the Indicator
Property: id
Type: String
Required: True

The dayIndicator object accepts an id property. This property expects a string value of the indicator element's ID.

let settings = {
  ...
  day: {
    id: 'day-indicator-disc'
  }
};
Day - Offset the Hours
Property: offsetHours
Type: Boolean
Required: False
Default: False

While most indicators rotate once a day to depict the current label, some update regularly with the time to also indicate the progression through the day. Using the offsetHours setting allows TickTock to not only rotate the element to the correct day but to also rotate an additional amount for the day's current hour.

let settings = {
  ...
  day: {
    id: 'day-indicator-disc',
    offsetHours: true
  }
};
Month - Overview
Property: month
Type: Object
Required: False
Default: null

Month indicators are used to display the current month of the year. This is commonly seen as a rotating dial with the month abbreviation being shown through a window but can also be built as a rotating hand pointing to the current month.

let settings = {
  ...
  month: {
    id: 'month-disc'
  }
};
Month - Define the Indicator
Property: id
Type: String
Required: True

The month object accepts an id property. This property expects a string value of the indicator element's ID. The indicator's initial position is expected to be on January and will rotate clockwise for each month from there.

let settings = {
  ...
  month: {
    id: 'month-disc'
  }
};
Year - Overview
Property: year
Type: Object
Required: False
Default: null

A year indicator is commonly included within perpetual calendar complications. It will indicate the current year's relation to the next leap year. Usually shown as four sections with a rotating hand, the indictor will highlight when the current year is a leap year and show the other year's position otherwise.

TickTock also accounts for designs that may want to show the year's progress. Much like the Day Indicator's offsetHours functionality, the offsetMonths functionality rotates the indicator 90deg for each year and an additional 7.5deg for each month. 

let settings = {
  ...
  year: {
    id: 'year-disc',
    offsetMonths: true
  }
};
Year - Define the Indicator
Property: id
Type: String
Required: True

The year object accepts an id property. This property expects a string value of the indicator element's ID. TickTock, by default, expects the initial position of the indicator pointing toward the middle of the first year in the set of four with the fourth year being the leap year.This is different if using offsetMonths.

let settings = {
  ...
  year: {
    id: 'year-disc'
  }
};
Year - Offset the Months
Property: offsetMonths
Type: Boolean
Required: False
Default: False

TickTock also accounts for designs that may want to show the year's progress. Much like the Day Indicator's offsetHours functionality, the offsetMonths functionality rotates the indicator 90deg for each year and an additional 7.5deg for each month.

TickTock expects the initial position of the indicator pointing toward the first year in the set of four with the fourth year being the leap year. If the indicator is not going to offset the months, the indictor can point to the middle of the year and will rotate 90deg for each additional year. But if the indicator is to offset the months, the initial position should begin at the very beginning of the first year label as it will rotate 90deg per year and an additional 7.5deg per month. 

let settings = {
  ...
  year: {
    id: 'year-disc',
    offsetMonths: true
  }
};

Day/Night Indicator

Overview
Property: dayNightIndicator
Type: Object
Required: False
Default: null

A day/night indicator shows which half the day cooresponds to. This does not sync up with sunrise and sunset but merely depicts AM and PM. This is a feature commonly found with 12-hour dials or dual time complications to more clearly convey the time of day.

Many simple day/night indicators are a two-color disc that show "day" only from midnight to 6 AM and "night" only from noon to 6 PM, with a half-day or half-night display the rest of the time. TickTock follows this convention.

let settings = {
  ...
  dayNightIndicator: {
    id: 'day-night-dial',
    dial: 1,
    invert: true
  }
};
Define the Indicator
Property: id
Type: String
Required: True

The dayNightIndicator object accepts an id property. This property expects a string value of the indicator element's ID. The default behavior will be for the indicator to rotate clockwise relative to the time of the first dial passed in the dials array. 

let settings = {
  ...
  dayNightIndicator: {
    id: 'day-night-dial'
  }
};
Target a Specific Dial
Property: dial
Type: Number
Required: False
Default: 0

The default behavior for the day/night indicator is to depict the time of the first dial (index 0) passed into the dials array. However, if the indicator should represent the time of a secondary dial, use the dial property and pass in a number cooresponding to the dials array index value.

Note that if changing the default dial for the indicator, the target dial will need to have a name property provided. The indicator looks for the custom dial by name when checking the time and updating.

In the example code below, the day/night indicator will be tied to the 'secondary' dial in the dials array by setting its dial property to a value of 1. 

let settings = {
  dials: [{
      name: 'primary',
      hands: {
        hour: 'dial-primary-hour-hand',
        minute: 'dial-primary-minute-hand',
        second: 'dial-primary-second-hand'
      }
    },
    {
      name: 'secondary',
      hands: {
        hour: 'dial-secondary-hour-hand',
        minute: 'dial-secondary-minute-hand',
        second: 'dial-secondary-second-hand'
      },
      offset: '+6',
      sweep: true
    }]
  dayNightIndicator: {
    id: 'day-night-dial',
    dial: 1
  }
};
Invert the Direction
Property: invert
Type: Boolean
Required: False
Default: False

By default, the day/night indicator rotates clockwise with the initial position set to the full day display. To invert this behavior, and rotate counter clockwise, set the invert property to true.

let settings = {
  ...
  dayNightIndicator: {
    id: 'day-night-dial',
    dial: 1,
    invert: true
  }
};

Dials

Overview

The dials are the primary component in TickTock. A dial serves as a collection of hand elements and settings used to indicate the time by rotating the elements according to the current local time, the provided GMT offset, or the manually set time. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    },
    offset: '+3',
    format: 12,
    sweep: true
  }]
};
Defining a Dial
Property: dials
Type: Array
Required: True

The dials property is an array that contains objects for each individual dial. This allows flexibility in how to display the time as well as how many time zones can be shown.

Every hand element entered in a dial object will work in unison regardless of where they visually appear in the design. For example, if the watch has large hour and minute hands but shows the seconds in a sub-dial at 6 o'clock, those can still be designated in the same dial object. Just because visually they are not together does not mean that they must be entrered as separate dials. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    },
    offset: '+3',
    format: 12,
    sweep: true
  }]
};
Dial Names
Property: name
Type: String
Required: False
Default: null

Dial names are not required in most instances. The name is primarily used for personal reference. However, the name does serve a purpose when including the Day/Night Indicator complication. Because Day/Night Indicators are often used on watches that have multiple dials, the name of the dial is used to link the indicator to the correct dial. 

let settings = {
  dials: [{
    name: 'primary-dial',
    ...
  }]
};
Defining Hands
Property: hands
Type: Object
Required: True

The hands object contains three properties, hour, minute, and second. Each property contains a string value of corresponding element's ID.

While the hands object is required for the dial component, not every hand is required. A dial could contain an hour and minute hand, just a second hand, or any combination of the three. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }]
};
GMT Offsets
Property: offset
Type: String
Required: False
Default: null

TickTock defaults to the local timezone by using JavaScript's native Date() object. However, a dial can be given a specific GMT offset value to override this default.

The offset property is a string supporting both negative and positive values. The values are relative to the Greenwich Mean Time (GMT). So to set a dial to show the time in New York City the offset property would have a value of '-5'. Whereas showing the time in Singapore would be '+8'.

Decimal values are also supported. For example, Delhi would have an offset value of '+5.3'. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }],
  offset: '+2' // Helsinki, Finland
};
12/24-hour Formats
Property: format
Type: Number
Required: False
Default: 12

Most watches show time in a 12-hour format where the hour hand rotates a full circle every 12 hours. But if the dial is to show the time in a 24-hour format, TickTock supports the format property. The property defaults to 12 but can be set to 24 to adust the rotation value of the hour hand.

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }],
  offset: '-4',
  format: 24
};
Sweeping Seconds
Property: sweep
Type: Boolean
Required: False
Default: False

By default, the second hand of each dial will tick or jump from one position to the next. However, by setting the sweep property to true TickTock will apply a CSS transition to the second hand to have it smoothly rotate around the dial.

The sweep property only affects the second hand of the dial. To have a sweeping motion for the minute and hour hands as well, a CSS transition will need to be added manually. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }],
  offset: '-4',
  format: 24,
  sweep: true
};
Multiple Dials

Because the dials property is an array of dial objects, a single watch can display multiple dials at once. For example, Dual Time watches which show both local and home times together. TickTock supports passing in multiple dial objects each with their own settings to support such complications.

let settings = {
  dials: [{
      name: 'primary',
      hands: {
        hour: 'dial-primary-hour-hand',
        minute: 'dial-primary-minute-hand',
        second: 'dial-primary-second-hand'
      }
    },
    {
      name: 'secondary',
      hands: {
        hour: 'dial-secondary-hour-hand',
        minute: 'dial-secondary-minute-hand',
        second: 'dial-secondary-second-hand'
      },
      offset: '+6',
      sweep: true
    }]
};

Manual Time (Crown)

Overview
Property: crown
Type: Object
Required: False
Default: null

The manual time complication allows a user to click the crown of the watch to temporarily pause its running. In this state, referred to as the set state, they're able to use the arrow keys to manually adjust the time. Upon clicking the crown again, TickTock will resume telling the time from the manually set position. To account for visual cues, on click, the crown will toggle an 'active' class.

Additionally, if a secondary dial is being used, while in the set state, the right arrow key will allow a user to target the secondary dial to set its time independently. 

let settings = {
  ...
  crown: {
    id: 'the-crown'
  }
};
Define the Crown
Property: id
Type: String
Required: True

The crown object accepts an id property. This property expects a string value of the crown element's ID. TickTock will automatically update the CSS cursor property of the crown element to pointer to visually indicate the functionality on hover.

let settings = {
  ...
  crown: {
    id: 'the-crown'
  }
};
Crown Active States

To allow for visual cues between states, the crown will toggle an 'active' class whenever clicked.

For example, this can allow a CSS transition to slide the crown out into a set position upon the first click and then transtion it back into its original position when clicked again.

Note that the enter key will execute the same functionality as clicking the crown based on the default keybindings of the Watch class.

Watch Keybindings

When the crown is clicked, a series of keydown event listeners are enabled to allow setting the time on dials, increasing power reserves, and alternating between dials. Additionally, the enter key is bound to replicate the behavior of clicking the crown.

Default Keybindings:

The enter key will toggle the crown between default and set states.

The left arrow key will wind the power reserve.

In Set State:

The up and down arrow keys will adjust the time of the active dial.

The right arrow key will alternate the active dial based on the order of the dials array.

Minute Repeater

Overview
Property: repeater
Type: Object
Required: False
Default: null

A repeater is a complication that audibly chimes the hours and often minutes at the press of a button. There are many types of repeater but the minute repeater chimes the time down to the minute, using separate tones for hours, quarter hours, and minutes.

The repeater button or trigger can be any element type as long as it contains an ID. Clicking the trigger will toggle playing the chimes. Upon playing the chimes, the time is stored so in the event a new minute passes after the initial button was clicked but before the chimes have completed, the latest minute will not be played. 

let settings = {
  ...
  repeater: {
    id: 'play-repeater',
    chimes: {
      hour: './dist/sounds/chime-01.mp4',
      quarter: './dist/sounds/chime-02.mp4',
      minute: './dist/sounds/chime-03.mp4'
    },
    dial: 1
  }
};
Define the Trigger
Property: id
Type: String
Required: True

The repeater object accepts an id property. This property expects the ID of the repeater trigger element as a string.

Clicking the trigger will toggle playing and stopping the chimes. Additionally, upon playing the chimes, the time is stored so in the event a new minute passes after the initial button was clicked but before the chimes have completed, the latest minute will not be played. 

let settings = {
  ...
  repeater: {
    id: 'repeater-btn'
  }
};
Define the Chimes
Property: chimes
Type: Object
Required: True

There are chimes included with TickTock, three separate sounds to represent minutes, quarter hours, and hours. These sounds were found on the Free Sound Project.

However, custom chimes can be used by defining the path to each audio file inside of the chimes object. 

let settings = {
  ...
  repeater: {
    id: 'repeater-btn',
    chimes: {
      hour: './dist/sounds/chime-01.mp4',
      quarter: './dist/sounds/chime-02.mp4',
      minute: './dist/sounds/chime-03.mp4'
    }
  }
};
Target a Specific Dial
Property: dial
Type: Number
Required: False
Default: 0

The minute repeater by default will chime the time of the first dial in the dials array. However, the repeater can read the time of secondary dials as well by passing the dial's array index into the dial property. 

let settings = {
  ...
  repeater: {
    id: 'play-repeater',
    chimes: {
      hour: './dist/sounds/chime-01.mp4',
      quarter: './dist/sounds/chime-02.mp4',
      minute: './dist/sounds/chime-03.mp4'
    },
    dial: 1
  }
};

Moonphase

Overview

Type: Object
Required: False

A moon-phase indication is a rotating disc, shown in a dial section, that indicates the position of the moon - usually the way it presents itself at the observation of the Northern Hemisphere.

TickTock will rotate the moonphase disc from new moon to waxing crescent, first quarter, waxing gibbous, full moon, waning gibbous, three quarter, and waning crescent. It will also default to rotating the disc clockwise beginning from the full moon position.

Type: Boolean
Required: False

TickTock will rotate the moonphase disc from new moon to waxing crescent, first quarter, waxing gibbous, full moon, waning gibbous, three quarter, and waning crescent. Power Reserve

Overview
Property: reserve
Type: Object
Required: False
Default: null

A power reserve indicator is a complication of the watch which is designed to show the amount of remaining stored energy. The power reserve indicator indicates the tension on the mainspring at any particular moment. If the indicator is fully drained, the watch will stop running until it's wound again.

Many reserve indicators are half circles or partial curves. So TickTock accepts a range of rotation values, its empty and full positions, to determine the movement of the hand. The hand will be placed in the full position by default and will drain 0.5deg per second. As the reserve reaches its empty range value, all complications cease to work until the watch is wound again by using the left arrow key.

Note that the range values are relative to the hand's starting position. 

let settings = {
  ...
  reserve: {
    id: 'power-reserve-hand',
    range: [-90, 90]
  }
};
Define the Element
Property: id
Type: String
Required: True
Default: null

The reserve object accepts an id property. This property expects the ID of the power reserve element as a string.

let settings = {
  ...
  reserve: {
    id: 'power-reserve-hand'
  }
};
Range of Motion
Property: range
Type: Array
Required: True

Many power reserve indicators are half or partial circles. Because of this, TickTock supports a range of motion for the reserve indicator. The range array should contain the min and max values of the indicator's range of motion.

The range of the indicator is relative to its original position. For example, if the reserve is shown as a half circle with the indicator drawn horizontally at 0 degrees, the full range of motion would be -90 degrees to 90 degrees.

TickTock expects that the first value in the range array is the minimum value. Upon reaching this value the watch will cease operating until wound again. TickTock then expects the second value in the range array to be the maximum rotation value. Winding the reserve will not exceed this value.

Upon initialization, the reserve indicator will be set to its max position and will drain 0.5 degrees every second. 

let settings = {
  ...
  reserve: {
    id: 'power-reserve-hand',
    range: [-90, 90]
  }
};

Watch

Overview

The Watch class serves as the parent container for all the child components. As settings are passed into the instance of the Watch, that instance is passed to each child component. This reference gives access to properties and methods that only exist on the Watch class.

Notably, the Watch class handles the initial key bindings and governs the overarching interval. The interval runs every second and updates the time and any other child component. If a watch contains a power reserve, the interval on the Watch instance will be cleared, stopping all watch functionality.

Default Keybindings

Every instance of the Watch class adds keydown event listeners for several keys. These events provide ways to interact with components of the watch such as the crown, adjusting time manually, and winding the power reserve.

Default Keybindings:

The enter key will toggle the crown between default and set states.

The left arrow key will wind the power reserve.

In Set State:

The up and down arrow keys will adjust the time of the active dial.

The right arrow key will alternate the active dial based on the order of the dials array.

Default Intervals

Every instance of the Watch class creates an interval to run every second. This interval will update the time and perform any checks and updates associated with other components.

For example, a watch with a day/night indicator will update the indicators position as the time reaches 12:00 AM because this check is part of the primary interval.

Watches that have a power reserve component will clear their interval when the reserve empties, or reaches its minimum range of motion, which will stop all watch functionality tied to the interview. This would leave the crown functionality as the only component that would still work with an empty reserve.

The interval can be stopped manually by calling instance.stopInterval() and resumed by calling instance.startInterval().

The right arrow key will alternate the active dial based on the order of the dials array.

Default Intervals

Every instance of the Watch class creates an interval to run every second. This interval will update the time and perform any checks and updates associated with other components.

For example, a watch with a day/night indicator will update the indicators position as the time reaches 12:00 AM because this check is part of the primary interval.

Watches that have a power reserve component will clear their interval when the reserve empties, or reaches its minimum range of motion, which will stop all watch functionality tied to the interview. This would leave the crown functionality as the only component that would still work with an empty reserve.

The interval can be stopped manually by calling instance.stopInterval() and resumed by calling instance.startInterval().
