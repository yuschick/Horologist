Dials

Overview

The dials are the primary component in TickTock. A dial serves as a collection of hand elements and settings used to indicate the time by rotating the elements according to the current local time, the provided GMT offset, or the manually set time. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    },
    offset: '+3',
    format: 12,
    sweep: true
  }]
};
Defining a Dial
Property: dials
Type: Array
Required: True

The dials property is an array that contains objects for each individual dial. This allows flexibility in how to display the time as well as how many time zones can be shown.

Every hand element entered in a dial object will work in unison regardless of where they visually appear in the design. For example, if the watch has large hour and minute hands but shows the seconds in a sub-dial at 6 o'clock, those can still be designated in the same dial object. Just because visually they are not together does not mean that they must be entrered as separate dials. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    },
    offset: '+3',
    format: 12,
    sweep: true
  }]
};
Dial Names
Property: name
Type: String
Required: False
Default: null

Dial names are not required in most instances. The name is primarily used for personal reference. However, the name does serve a purpose when including the Day/Night Indicator complication. Because Day/Night Indicators are often used on watches that have multiple dials, the name of the dial is used to link the indicator to the correct dial. 

let settings = {
  dials: [{
    name: 'primary-dial',
    ...
  }]
};
Defining Hands
Property: hands
Type: Object
Required: True

The hands object contains three properties, hour, minute, and second. Each property contains a string value of corresponding element's ID.

While the hands object is required for the dial component, not every hand is required. A dial could contain an hour and minute hand, just a second hand, or any combination of the three. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }]
};
GMT Offsets
Property: offset
Type: String
Required: False
Default: null

TickTock defaults to the local timezone by using JavaScript's native Date() object. However, a dial can be given a specific GMT offset value to override this default.

The offset property is a string supporting both negative and positive values. The values are relative to the Greenwich Mean Time (GMT). So to set a dial to show the time in New York City the offset property would have a value of '-5'. Whereas showing the time in Singapore would be '+8'.

Decimal values are also supported. For example, Delhi would have an offset value of '+5.3'. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }],
  offset: '+2' // Helsinki, Finland
};
12/24-hour Formats
Property: format
Type: Number
Required: False
Default: 12

Most watches show time in a 12-hour format where the hour hand rotates a full circle every 12 hours. But if the dial is to show the time in a 24-hour format, TickTock supports the format property. The property defaults to 12 but can be set to 24 to adust the rotation value of the hour hand.

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }],
  offset: '-4',
  format: 24
};
Sweeping Seconds
Property: sweep
Type: Boolean
Required: False
Default: False

By default, the second hand of each dial will tick or jump from one position to the next. However, by setting the sweep property to true TickTock will apply a CSS transition to the second hand to have it smoothly rotate around the dial.

The sweep property only affects the second hand of the dial. To have a sweeping motion for the minute and hour hands as well, a CSS transition will need to be added manually. 

let settings = {
  dials: [{
    name: 'primary-dial',
    hands: {
      hour: 'element-id',
      minute: 'element-id',
      second: 'element-id'
    }
  }],
  offset: '-4',
  format: 24,
  sweep: true
};
Multiple Dials

Because the dials property is an array of dial objects, a single watch can display multiple dials at once. For example, Dual Time watches which show both local and home times together. TickTock supports passing in multiple dial objects each with their own settings to support such complications.

let settings = {
  dials: [{
      name: 'primary',
      hands: {
        hour: 'dial-primary-hour-hand',
        minute: 'dial-primary-minute-hand',
        second: 'dial-primary-second-hand'
      }
    },
    {
      name: 'secondary',
      hands: {
        hour: 'dial-secondary-hour-hand',
        minute: 'dial-secondary-minute-hand',
        second: 'dial-secondary-second-hand'
      },
      offset: '+6',
      sweep: true
    }]
};
